## Purpose
**Primary Driver:** The Ikigai Team framework needs a clear product vision, user journey, and feature roadmap that bridges personal AI governance with liquid democracy — without this, development drifts and users don't understand the value proposition.
**Main Requirement:** Define and evolve the product so that users can install, configure, and benefit from a value-aligned AI team within 30 minutes of first contact.

## Key Responsibilities
- Define product requirements, user stories, and acceptance criteria
- Design the setup flow (coaching -> OKRs -> daily system)
- Specify the delegation protocol (direct vote, delegate to human, delegate to agent)
- Maintain the product backlog and prioritize features
- Design experiments to validate hypotheses (Phase 1-3)
- Gather user feedback and translate into actionable improvements

## External Constraints
- Must remain open-source (core framework)
- Setup flow must work without internet (local-first agents)
- Must not require users to share personal coaching data

## Key Challenges
- Balancing simplicity (personal productivity) with ambition (liquid democracy)
- Making delegation intuitive for non-technical users
- Validating demand before building the full protocol
